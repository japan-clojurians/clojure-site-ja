ネームスペース

Table of Contents
  • 関連する関数

ネームスペースは単純な(未装飾の)シンボルからVar、クラスへのマッピングを定義する。Varは def やその派生を使ってネームスペースにインターンすることが出来る。その場合Varは単純な名前としてのシンボルとそれを含むネームスペースの参照及び同じvarを示すネームスペースマップを持つ。ネームスペースは refer や use を使用して他のネームスペースにインターンされたVarに対するマッピングを含むことや、 import を使用してクラスオブジェクトのシンボルに対するマッピングを含むことが出来る。ネームスペースはファーストクラスなので列挙すること等が出来る。加えてネームスペースは動的なのでランタイムにRepl等で作成、変更、削除を行うことが出来る。

Clojureのソースファイルの一番始めにネームスペースを定義する最良の方法は ns マクロ を使用することだ。これはデフォルトでは java.lang と clojure.lang.Compiler のクラス名及び clojure.core 内の関数へのマッピングを含むネームスペースを新たに作成する。

Replからは in-ns を使用するのがベストで、新しく作られたネームスペースには java.lang のクラス名のマッピングのみが含まれる。 clojure.core のネームスペースにある名前アクセスするには (clojure.core/refer 'clojure.core) を実行する必要がある。Replにおける user ネームスペースではすでにこれが実行されている。

現在のネームスペース、 *ns* は in-ns 、もしくは ns マクロ で設定することが出来、またこれらの方法でのみ作られるべきである。どちらも指定されたネームスペースが存在しない場合、新たにそのネームスペースを作成する。

ns-name

ns-interns
ns-map
ns-publics
ns-refers

resolve

namespace

ns-unalias
ns-unmap
remove-ns
